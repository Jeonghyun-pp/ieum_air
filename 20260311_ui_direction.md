## 1. 현재 UI/UX 진단

### 1.1 랜딩 페이지 현황

```
현재 구조 (7개 섹션):
  Hero → Services → HowItWorks → LiveDemo → Pricing → FAQ → FinalCTA

강점:
  ✓ 다크 테마 + 보라 그래디언트 — 세련된 첫인상
  ✓ framer-motion 애니메이션 — 부드러운 스크롤 경험
  ✓ LiveDemo (가격 캘린더 인터랙션) — 서비스 직관적 이해
  ✓ 반응형 레이아웃 — 모바일/데스크탑 대응

문제점:
  ✗ "에이전시 서비스" 톤 — "우리가 대신 해드립니다" 느낌
     → 현실: 파이프라인 적용 후에는 "데이터가 알려줍니다" 모델
  ✗ Hero 통계가 모호 — "점유율 79%", "6개 서비스"
     → 구체적 가치 전달 부족 (경쟁자 대비 무엇을 개선하는지)
  ✗ Services 6개가 "제작 대행" 중심
     → 리스팅 최적화, SNS 콘텐츠, 블로그, 번역, 채널 전략, 가격 캘린더
     → 파이프라인의 핵심 가치(비교군 분석, 편의시설 격차, AI 진단)가 빠져 있음
  ✗ 가격 체계가 "월정액 대행료" 구조 (20만/35만/55만원)
     → SaaS 전환 시 가격 모델도 변경 필요
  ✗ 사회적 증거 부족 — 고객 사례, 리뷰, 실제 데이터 없음
```

### 1.2 포털 현황

```
현재 구조 (8개 페이지):
  Dashboard → Plan → Pricing → Content → Analytics → Assets → Results → Settings

강점:
  ✓ Spotify 스타일 다크 UI — 일관된 디자인 시스템
  ✓ PortalContext + usePortalData — 깔끔한 데이터 흐름
  ✓ 실시간 데이터 수집 파이프라인 (Instagram/GA4 크론)
  ✓ Skeleton 로딩 — 좋은 로딩 UX

문제점:
  ✗ 대부분 Mock 데이터 — 실사용자에게 보여줄 실제 데이터가 부족
  ✗ "관리형 서비스" UI — 할 일 목록, 콘텐츠 납품, 플랜 승인
     → 호스트가 "수동적 클라이언트"가 되는 구조
  ✗ 핵심 부재: 경쟁자 대비 내 숙소 현황
     → "비교군 75개 중 상위 45%" 같은 인사이트가 없음
  ✗ Analytics 페이지 — GA4/Instagram 데이터만
     → Airbnb 검색 순위, 가격 포지션, 편의시설 격차 없음
  ✗ Pricing 캘린더 — 이벤트 기반 추천만
     → 비교군 가격 대비 내 위치가 보이지 않음
  ✗ Content 페이지 — 콘텐츠 납품물 목록
     → 리스팅 제목/설명 최적화 제안이 없음
```

  ✗ 성과 추적 (Stage 4)

## 2. 핵심 과제: 기존 강점 유지 + 데이터 근거 강화

### 2.1 현재 UI의 강점 (반드시 유지할 것)

```
현재 포털이 잘하는 것 = "가이드형 UX"

  ✓ 대시보드 할 일 목록
    → 체크박스 + 기한 + "필수" 태그
    → 호스트가 뭘 해야 하는지 한눈에 파악
    → 완료 체크 시 성취감

  ✓ 이번 달 하이라이트
    → "가격 조정 3건 추천", "릴스 2개 제작 완료"
    → 색상 코드로 카테고리 구분 (초록/핑크/파랑)
    → 시각적 우선순위 전달

  ✓ 전략 요약 + 이유 배지
    → "왜 이걸 해야 하는지" 한 줄 요약
    → 이유 태그가 근거를 제시

  ✓ 가격 캘린더의 "적용했어요" 버튼
    → 추천 → 행동의 직관적 연결
    → 호스트의 능동적 참여 유도

  ✓ 콘텐츠 납품물 확인/다운로드
    → "무엇이 만들어졌는지" 시각적 확인
    → 대행 서비스의 실체감 제공

  ✓ 전략 플랜 승인/리비전
    → 호스트가 의사결정에 참여
    → "일방적 실행"이 아닌 "합의 후 실행"

이것들은 버리는 것이 아니라, 데이터 근거를 입혀서 더 강하게 만든다.
```

### 2.2 현재 UI에 부족한 것

```
현재 UI의 한계 = "근거 없는 제안"

  현재:  "셀프 체크인 추가하세요" (왜?)
  목표:  "셀프 체크인 추가하세요"
          — 비교군 89%가 보유, 상위 성과자 100%
          — 추가 시 Health Score +5 예상
          [완료 체크 ✓]

  현재:  "가격 +20% 조정 추천" (근거?)
  목표:  "가격 +20% 조정 추천"
          — 비교군 중앙값 ₩96,000 (현재 ₩80,000)
          — 벚꽃축제 기간 비교군 평균 +25%
          [적용했어요 ✓]

  현재:  "인스타 릴스 2개 제작 완료" (효과?)
  목표:  "인스타 릴스 2개 제작 완료"
          — 지난달 릴스 게시 후 프로필 방문 +32%
          — 일본 게스트 유입 +15%
          [다운로드] [성과 보기]

  → "무엇을 하세요" (기존 강점) + "왜냐하면" (데이터 근거) = 최종 목표
```

### 2.3 해결 방향: "데이터 기반 가이드" 하이브리드

```
┌─────────────────────────────────────────────────────────────┐

## 3. 제안 방향: "데이터 인텔리전스 포털"

### 3.1 핵심 UX 원칙

```
① 첫 화면에서 "내 숙소의 위치"를 보여준다
   → Health Score 게이지 + 비교군 대비 백분위
   → 호스트가 즉시 "나는 상위 몇 %인가" 파악

② 모든 페이지에 "비교군 기준선"을 함께 보여준다
   → 가격 캘린더: 내 가격 + 비교군 중앙값 오버레이
   → 분석: 내 평점 vs 비교군 평균
   → 단독 수치 대신 항상 "대비" 수치

③ "무엇을 해야 하는지" 구체적으로 제안한다
   → "편의시설 6개 추가하면 상위 30%" (체크리스트)
   → "주중 가격 ₩70,000으로 낮추면 점유율 +15%" (원클릭)
   → "제목을 이렇게 바꾸면 검색 노출 +20%" (AI 제안 3개)

④ 행동 → 결과를 연결한다
   → "지난달 편의시설 3개 추가 → Health Score 45→52 상승"
   → 최적화 이벤트 로그 + 효과 트래킹
```

### 3.2 새로운 페이지 구조

```
현재 (8페이지):                    제안 (10페이지):
─────────────────                  ─────────────────
Dashboard (대시보드)        →      Dashboard (경쟁력 + 가이드)  ★ 강화
Plan (전략)                →      Plan (전략)                  유지 (Layer 3)
Pricing (가격 캘린더)       →      Pricing (가격 인텔리전스)    ★ 강화
Content (콘텐츠 납품)       →      Content (콘텐츠 납품)        유지 (Layer 3)
Analytics (GA4/Instagram)  →      Analytics (통합 분석)        ★ 강화
Assets (할일/파일/메시지)   →      Assets (할일/파일/메시지)    유지 (가이드 허브)
Results (성과 리포트)       →      Results (효과 추적)          ★ 강화
Settings (연동)            →      Settings (연동)              유지
                                  Competitors (비교군 분석)    ★ 신규
                                  Optimize (리스팅 최적화)     ★ 신규

핵심 원칙:
  - 기존 페이지를 제거하지 않는다
  - 기존 페이지에 "데이터 근거"를 입힌다
  - 신규 2개 (Competitors, Optimize)는 파이프라인 데이터 표시용
  - Plan/Content/Assets는 대행 서비스(Layer 3) 고객용으로 유지
```

### 3.3 네비게이션 변경

```
현재 사이드바:                     제안 사이드바:
─────────────                     ─────────────

                                   ── 인텔리전스 ──
🏠 Home                           🏠 대시보드              ← 강화
                                   📊 비교군 분석           ← 신규
                                   ✏️ 리스팅 최적화         ← 신규
💰 Pricing Calendar                💰 가격 인텔리전스       ← 강화
📊 Analytics                       📈 분석                 ← 강화
📈 Results                         📋 효과 추적            ← 강화

                                   ── 대행 서비스 ── (Layer 3 구독자에게만 표시)
📋 Plan                            📋 전략
📸 Content                         📸 콘텐츠
📦 Assets                          📦 Assets

⚙️ Settings                        ⚙️ 설정

→ 사이드바를 "인텔리전스"와 "대행 서비스" 섹션으로 구분
→ 무료/SaaS 사용자: 인텔리전스 섹션만 표시
→ 대행 구독자: 양쪽 모두 표시
→ 기존 기능을 숨기는 것이 아니라, 맥락에 맞게 그룹핑
```

## 4. 랜딩 페이지 리디자인

### 4.1 메시지 전환

```

  "숙소 매출, 데이터로 올립니다"
  "데이터 기반 숙소 홍보 서비스"
  → 모호하고 에이전시 느낌

제안 메시지:
  "내 숙소는 경쟁자 대비 몇 등일까?"
  "주변 75개 숙소와 비교해서 알려드립니다"
  → 호기심 유발 + 구체적 가치 전달
```

### 5.4 Optimize (리스팅 최적화) — 신규 페이지 (Content는 별도 유지)

#### Content 페이지: 기존 그대로 유지
```
기존 Content 페이지 (Instagram/TikTok/Blog 납품물 그리드)는
대행 서비스(Layer 3) 구독자용으로 그대로 유지한다.

  → 탭 필터 (전체/인스타/틱톡/블로그)
  → 콘텐츠 카드 그리드
  → 상세 모달 (다운로드/보기/코멘트)
  → 이 구조는 변경하지 않음
