## 1. 외부 데이터 소스 총정리

### A. 이벤트/행사 데이터 (가격 조정 근거)

| 소스 | API 엔드포인트 | 비용 | 데이터 내용 | 신선도 |
|------|---------------|------|-----------|--------|
| 한국관광공사 TourAPI | `apis.data.go.kr/B551011/KorService2/searchFestival2` | 무료 (10만건/일) | 지역별 축제/행사, 날짜, 장소, GPS 좌표 | 실시간 |
| KOPIS 공연정보 | `kopis.or.kr/openApi/restful/pblprfr` | 무료 (별도가입 필요) | 콘서트, 뮤지컬, 공연 일정, 장소, 장르, 티켓 판매 | 실시간 |
| 공휴일 API | `apis.data.go.kr/B090041/.../SpcdeInfoService/getRestDeInfo` | 무료 (1만건/일) | 공휴일, 국경일, 기념일, 24절기 | 연간 |
| 전국문화축제 표준데이터 | `api.data.go.kr/openapi/tn_pubr_public_cltur_fstvl_api` | 무료 | 문화축제명, 기간, 장소, 좌표, 주관기관 | 분기 갱신 |
| 한국문화정보원 | `apis.data.go.kr/B553457/cultureinfo/period2` | 무료 (1천건/일) | 전시, 공연, 문화행사 통합 조회 | 실시간 |

#### TourAPI 상세 (핵심 소스)
- **가입:** data.go.kr 회원가입 → "한국관광공사 국문 관광정보" 활용신청 → 즉시~1영업일 승인
- **주요 파라미터:** `eventStartDate`, `eventEndDate`, `areaCode` (1=서울, 6=부산, 39=제주 등), `contentTypeId=15` (축제/행사)
- **응답 필드:** 행사명, 시작/종료일, 장소, GPS, 포스터 이미지
- **data.go.kr API 키 하나로 공휴일/문화축제 등 다른 공공 API도 공유 사용 가능**

#### KOPIS 상세
- **가입:** kopis.or.kr에서 별도 가입 (data.go.kr과 별개)
- **응답 형식:** XML만 지원 (JSON 미지원)
- **주요 파라미터:** `stdate`, `eddate`, `signgucode` (지역코드), `genrecode` (AAAA=뮤지컬, BBBB=콘서트 등), `prfstate` (01=예정, 02=공연중)
- **데이터:** 인터파크, YES24, 멜론티켓 등 주요 티켓 플랫폼 데이터 통합

### B. 시장/경쟁 데이터 (가격 포지셔닝)

| 소스 | 방법 | 비용 | 데이터 내용 | 비고 |
|------|------|------|-----------|------|
| Airbnb 주변 숙소 | Puppeteer/Playwright 스크래핑 | 인프라비용만 | 가격, 평점, 리뷰수, 가용성 | 속도 제한 필요 (3~5초/건) |
| Google Hotels | SerpAPI (`engine=google_hotels`) | 무료 250건/월, $25/월 | 호텔/숙소 가격, 평점, 위치 | Booking/Agoda 가격 포함 |
| Apify Airbnb Actor | Apify 매니지드 스크래핑 | ~$10/월 | 경쟁 숙소 캘린더, 점유율 추정 | 안정적, 유지보수 불필요 |
| AirDNA | 전문 STR 데이터 | $25~100+/월 | 점유율, ADR, RevPAR, 시장 분석 | 가장 정확, 고비용 |

#### Airbnb 스크래핑 상세
- **검색 URL 구조:** `airbnb.com/s/Jeju/homes?checkin=...&checkout=...&ne_lat=...&sw_lng=...`
- **추출 가능 데이터:** 숙소명, 1박 가격, 총 가격, 평점, 리뷰수, 슈퍼호스트 여부
- **점유율 추정:** 캘린더에서 blocked 날짜 = 예약 또는 차단 → 점유율 근사치
- **리뷰 속도:** 월별 신규 리뷰 수 ≈ 점유율 상관관계
- **주의사항:** Airbnb TOS 위반 가능, 인간 속도 스크래핑(3~5초/건), 개인정보 미저장, 시장분석 목적만

#### Google Hotels via SerpAPI
- **파라미터:** `q=hotels in Jeju`, `check_in_date`, `check_out_date`, `currency=KRW`, `vacation_rentals=true`
- **장점:** Booking.com, Agoda 등 OTA 가격이 Google에 노출된 것을 일괄 수집
- **SerpAPI 하나로 Google Trends + Google Hotels 동시 사용 가능 (동일 크레딧)**

#### Google Trends via SerpAPI 상세
- **핵심 활용:** `q=Jeju+accommodation&geo=JP&data_type=TIMESERIES` → 일본에서 "제주 숙소" 검색량 추이
- **국가 코드:** JP(일본), TW(대만), US(미국), CN(중국), HK(홍콩), TH(태국), SG(싱가포르)
- **데이터 타입:** `TIMESERIES` (시계열), `GEO_MAP` (지역분포), `RELATED_QUERIES` (연관검색어)
- **대안:** searchapi.io도 유사 서비스 제공

#### 법무부 입국자 통계 상세
- **엔드포인트:** `https://api.odcloud.kr/api/15099989/v1/` (연도별 UUID 존재)
- **응답 필드:** 연도, 월, 국적지역, 입국자수
- **갱신 주기:** 월 1회 (약 30일 지연)
- **주의:** 순수 방문자 수가 아닌 입국 건수 (재방문 중복 포함)

#### 관광빅데이터 상세
- **데이터 원천:** KT (내국인) + SK텔레콤 (외국인) 모바일 통신사 데이터
- **응답:** 기준일, 지역명, 요일, 방문자유형, 방문자수
- **지연:** 약 2개월 (통신사 배치 처리)

#### 환율 API 상세
- **Fawaz Currency API (1순위):** CDN 기반, 인증 불필요, 200+ 통화, 일별 갱신
- **ExchangeRate-API (백업):** `open.er-api.com/v6/latest/USD`, 무료 1,500건/월
- **활용 통화:** JPY(일본), TWD(대만), CNY(중국), USD(미국), HKD(홍콩), THB(태국)

#### Amadeus 항공 상세
- **TypeScript SDK:** `npm install amadeus` (커뮤니티: `amadeus-ts`)
- **주요 API:** Airport Routes (공항별 취항 노선), Flight Offers Search (실시간 가격), Flight Price Analysis (가격 이력)
- **한국 공항 코드:** CJU(제주), ICN(인천), GMP(김포)
- **샌드박스:** 고정 데이터셋, 월 ~2,000건/API, 프로덕션 전환 시 유료

## 2. 분석 파이프라인 설계

### 아웃풋 1: 가격 최적화 캘린더

```
[입력 데이터]                    [분석 로직]                     [고객 아웃풋]

TourAPI 축제 ──┐
KOPIS 공연 ────┤
공휴일 ────────┤── 이벤트 통합 ──┐
문화행사 ──────┘                 │
                                ├── 날짜별 가격     ──→  가격 캘린더
경쟁숙소 가격 ─┐                │   조정률 산출              (+50%, +30% 등)
Google Hotels ─┤── 시장가 분석 ──┤
               │                │                          근거 텍스트:
환율 ──────────┤── 수요 예측 ───┘                          "BTS 콘서트 (3/15)
입국자 통계 ───┤                                            + 벚꽃축제 기간
Google Trends ─┘                                            + 일본 검색량 180%↑
                                                            → +50% 권장"
날씨 예보 ─────── 보조 보정 ────┘
```

#### 가격 조정률 산출 로직

**Step 1: 기본 이벤트 조정률**

| 이벤트 타입 | 기본 조정률 | 비고 |
|------------|-----------|------|
| 대형 콘서트/공연 | +40~60% | KOPIS 공연 규모(객석수)에 따라 |
| 축제 | +20~40% | TourAPI 축제 기간 |
| 공휴일/연휴 | +15~30% | 연휴 길이에 비례 |
| 지역 소규모 행사 | +10~20% | 문화정보원 데이터 |
| 이벤트 복합 (겹침) | 가산 적용 | 최대 cap 설정 필요 |

**Step 2: 시장가 보정**
- 주변 경쟁 숙소 평균가 계산 (유사 타입/규모 필터)
- 현재 가격 ÷ 평균가 = 포지션 비율
- 포지션 비율 < 0.8 → 인상 여유 크다 → 이벤트 조정률 유지/상향
- 포지션 비율 > 1.2 → 이미 비싸다 → 이벤트 조정률 하향

**Step 3: 수요 보정 계수**
- 국적별 입국자 전월비 증감률
- Google Trends 검색량 변화율
- 환율 유불리 (강세 = 유리, 약세 = 불리)
- 3가지 종합 → 수요 계수 (0.8 ~ 1.3)
- 최종 조정률 = 기본 조정률 × 시장 보정 × 수요 계수

**Step 4: 기상 보정 (선택적)**
- 악천후 예보 (태풍, 폭우) → 조정률 소폭 하향 (-5~10%)
- 맑은 날씨 지속 → 유지

#### 고객에게 보이는 아웃풋
- 월간 캘린더에 날짜별 색상 표시 (초록: 일반, 노랑: +15~30%, 빨강: +40%+)
- 각 날짜 클릭 시 근거 표시:
  - 어떤 이벤트가 있는지
  - 주변 숙소 대비 가격 포지션
  - 수요 지표 요약
  - 최종 추천 가격

### 아웃풋 2: 월간 마케팅 전략

```
[입력 데이터]                    [분석 로직]                     [고객 아웃풋]

입국자 통계 ───┐
GA4 국적 ──────┤── 국적별        타겟 국가 순위
Google Trends ─┤   수요 분석 ──→ "이번달 1순위: 일본"
환율 ──────────┘                 + 근거 데이터

항공 노선 ─────┐
검색 트렌드 ───┤── 채널/매체     추천 채널 + 콘텐츠 방향
Instagram ─────┤   추천 ───────→ "일본 타겟 → Instagram
GA4 유입채널 ──┘                  릴스, 일본어 해시태그"

경쟁숙소 ──────┐
Airbnb 평점 ───┤── 경쟁 포지셔닝 차별화 포인트 도출
리뷰 데이터 ───┘   분석 ───────→ "평점 4.9 (상위 5%),
                                  가격은 평균 대비 저렴"
```

#### 타겟 국가 선정 로직

**국가별 종합 점수 산출:**

```
점수(국가) = 입국자증감률(w=0.3) × 검색트렌드(w=0.3) × 환율유불리(w=0.2) × GA4비율(w=0.2)
```

| 지표 | 가중치 | 데이터 소스 | 계산 방법 |
|------|--------|-----------|----------|
| 입국자 증감률 | 30% | 법무부 입국자 통계 | (이번달 - 전월) ÷ 전월 × 100 |
| 검색 트렌드 | 30% | Google Trends (SerpAPI) | 최근 4주 평균 ÷ 이전 4주 평균 |
| 환율 유불리 | 20% | Fawaz Currency API | 환율 변동률 (상대국 통화 강세 = 유리) |
| GA4 기존 비율 | 20% | GA4 Analytics | 국적별 방문 비율 (기존 유입 기반) |

- 상위 3개국 = 이번달 타겟 국가
- 근거 텍스트 자동 생성: "대만 입국자 +25%, '제주' 검색량 +180%, TWD 강세 → 1순위 추천"

#### 채널 추천 로직

**국가별 주요 채널 매핑 (현재 하드코딩 → 점진적 데이터 기반 전환):**

| 국가 | 1차 채널 | 2차 채널 | 3차 채널 |
|------|---------|---------|---------|
| 일본 | Instagram | Google 검색 | YouTube |
| 대만 | Instagram | Facebook | 블로그 (Pixnet) |
| 미국 | Google 검색 | Instagram | YouTube |
| 중국 | 샤오홍슈 | 웨이보 | Ctrip |
| 홍콩 | Instagram | Facebook | Google |
| 태국 | Facebook | Instagram | TikTok |
| 싱가포르 | Google | Instagram | TripAdvisor |
| 한국 | 네이버 | Instagram | YouTube |

- GA4 유입 채널 실적 데이터로 매핑 정확도 검증
- 항공 신규노선 취항 → 해당 국가 중요도 보너스

#### 콘텐츠 방향 추천 로직
- 이벤트 + 타겟 국적 조합 → 콘텐츠 주제 도출
  - 예: "3월 벚꽃축제 + 일본 타겟 → 벚꽃 뷰 강조 콘텐츠, 일본어 해시태그"
  - 예: "BTS 콘서트 + 대만 타겟 → 콘서트 접근성 강조, 중국어 캡션"
- Instagram 성과 데이터 → 어떤 유형의 콘텐츠가 효과적인지 피드백

#### 고객에게 보이는 아웃풋
- 타겟 국가 TOP 3 카드 (국기 + 점수 + 근거)
- 국가별 추천 채널 테이블 (채널 + 우선순위 + 예상 효과)
- 이번달 콘텐츠 방향 요약 (주제 + 톤앤매너 + 해시태그 제안)
- 전월 대비 지표 변화 (입국자, 검색량, 환율 추이 차트)

## 3. 데이터 수집 스케줄 설계

| 주기 | 수집 대상 | Cron 시간 | 저장 위치 |
|------|----------|----------|----------|
| 매일 | Instagram 인사이트 | 02:00 AM | `properties/{id}/analytics/` |

## 4. 신규 Firestore 컬렉션 설계

### `market-data/exchange-rates/{date}`
```typescript
{
  date: string             // "2026-03-04"
  rates: {
    JPY: number            // 1 KRW = ? JPY
    TWD: number
    CNY: number
    USD: number
    HKD: number
    THB: number
  }
  collectedAt: Timestamp
}
```

### `market-data/weather/{region}/{date}`
```typescript
{
  date: string
  region: string           // "jeju", "seoul", "busan"
  forecast: {
    tempMax: number
    tempMin: number
    precipitationMm: number
    weatherCode: number    // WMO code
    windSpeedMax: number
  }
  collectedAt: Timestamp
}
```

### `market-data/immigration/{month}`
```typescript
{
  month: string            // "2026-02"
  countries: Array<{
    country: string        // "일본", "대만", "미국"
    arrivals: number
    previousMonth: number
    changeRate: number     // 전월비 증감률 %
  }>
  collectedAt: Timestamp
}
```

### `market-data/trends/{month}`
```typescript
{
  month: string
  queries: Array<{
    keyword: string        // "제주 숙소", "Jeju accommodation"
    country: string        // "JP", "TW", "US"
    avgInterest: number    // 최근 4주 평균
    prevAvgInterest: number // 이전 4주 평균
    changeRate: number     // 증감률 %
  }>
  collectedAt: Timestamp
}
```

### `events-calendar/{eventId}` (기존 컬렉션, 스키마 확장)
```typescript
{
  title: string            // "BTS WORLD TOUR 부산"
  source: 'tourapi' | 'kopis' | 'holiday' | 'manual'
  sourceId: string         // 원본 API의 ID
  type: 'concert' | 'festival' | 'holiday' | 'sport' | 'exhibition' | 'local'
  region: string           // "busan", "jeju", "seoul"
  startDate: string        // "2026-03-15"
  endDate: string          // "2026-03-16"
  venue: string            // "부산 아시아드 주경기장"
  expectedScale: 'large' | 'medium' | 'small'  // 예상 규모
  baseAdjustment: number   // 기본 가격 조정률 (0.4 = +40%)
  gpsLat: number
  gpsLng: number
  collectedAt: Timestamp
}
```

### `properties/{id}/competitors/{competitorId}`
```typescript
{
  listingUrl: string
  title: string
  pricePerNight: number
  currency: string
  rating: number
  reviewCount: number
  propertyType: string
  distance: number         // 자사 숙소와의 거리 (km)
  collectedAt: Timestamp
}
```

## 5. 구현 우선순위

### Phase 1: 무료 API 즉시 연동 (1~2주)

| 순위 | 소스 | 이유 | 난이도 | 비용 |
|------|------|------|--------|------|
| 1 | TourAPI + KOPIS + 공휴일 | 가격 캘린더의 핵심 입력, 전부 무료 | 낮음 | 무료 |
| 2 | 환율 API (Fawaz) | 인증 불필요, 즉시 사용 가능 | 매우 낮음 | 무료 |
| 3 | 법무부 입국자 통계 | 국적별 수요의 "팩트" 근거 | 낮음 | 무료 |
| 4 | Open-Meteo 날씨 | 인증 불필요, 보조 지표 | 매우 낮음 | 무료 |

### Phase 2: 분석 로직 구현 (2~3주)

| 순위 | 작업 | 내용 |
|------|------|------|
| 5 | 이벤트 → 가격 조정률 산출 | Phase 1 데이터 기반 가격 추천 로직 |
| 6 | 국적별 수요 점수 산출 | 입국자 + 환율 → 타겟 국가 순위 |
| 7 | events-calendar 자동 적재 | TourAPI/KOPIS → Firestore 크론 |
| 8 | 기존 가격 캘린더 실데이터 연동 | 목데이터 → events-calendar 연결 |

### Phase 3: 유료 API 연동 (3~4주)

| 순위 | 소스 | 이유 | 비용 |
|------|------|------|------|
| 9 | Google Trends (SerpAPI) | 검색 수요 = 가장 선행적 지표 | $25/월 |
| 10 | Google Hotels (SerpAPI) | 경쟁 가격 포지셔닝 | 동일 크레딧 |
| 11 | Amadeus 항공 (샌드박스) | 공급 측면 보조 지표 | 무료 |

| 순위 | 작업 | 내용 |
|------|------|------|
